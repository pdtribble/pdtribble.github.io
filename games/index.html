overview

GameCenter is a self-hosted multiplayer gaming platform that I run on my homelab infrastructure — a Proxmox server and a Raspberry Pi. It supports real-time multiplayer games through a persistent WebSocket layer (Socket.io) and a shared SQLite database for lobby state, user sessions, and game history.

The architecture is deliberately modular: adding a new game is as simple as dropping a new module into the games/ directory and registering it with the router. No framework magic — just clean separation of concerns.

infrastructure

  • Proxmox VE for VM/container orchestration and snapshots
  • Raspberry Pi as an always-on Node.js server with nginx reverse proxy
  • SSL via Let's Encrypt with automatic renewal
  • Persistent game state in SQLite with WAL mode for concurrent reads
  • Systemd service for automatic restart on failure
  • Sync mechanism between Proxmox and Pi deployments

key features

  • Real-time multiplayer via Socket.io with room-based game isolation
  • Lobby system: create, join, and start games with a shareable code
  • Modular game architecture — each game is a self-contained module
  • Bot / AI player support for solo practice or uneven lobbies
  • Persistent leaderboards and match history per game
  • Responsive UI — playable on desktop and mobile
  • Turn management and timer logic per game type
