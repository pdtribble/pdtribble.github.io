ROVEROS

A WiFi-controlled robot platform built on the ESP32 microcontroller, with a real-time browser-based interface, block programming support, and hardware-aware battery monitoring.

View on GitHub

overview

ROVEROS is a WiFi-controlled robot built around the ESP32 microcontroller. The robot serves its own web interface directly from the chip — no app install, no Bluetooth pairing, no external server. You connect to its WiFi network, open a browser, and drive.

The control interface uses WebSockets for low-latency bidirectional communication. A virtual joystick on the web UI sends movement commands in real time, and the ESP32 translates them into PWM signals for the L298N dual H-bridge motor driver.

hardware

  • ESP32 development board (dual-core Xtensa LX6, built-in WiFi)
  • L298N dual H-bridge motor driver module
  • Two DC gear motors for differential drive
  • LiPo battery with voltage divider for ADC-based monitoring
  • Custom chassis (3D-printed + laser cut acrylic)
  • Optional: HC-SR04 ultrasonic sensor for obstacle avoidance

key features

  • Real-time joystick control via WebSockets served from the ESP32 itself
  • Battery voltage monitoring with L298N voltage drop compensation (~2V)
  • Block programming interface — drag-and-drop sequences of commands
  • Speed control via PWM with configurable dead-zone tuning
  • Access Point mode: robot creates its own WiFi network on boot
  • OTA (Over-the-Air) firmware updates via browser
  • Serial debug output for development and calibration

technical notes

The firmware is written in C++ using the Arduino framework for ESP32. The web server runs on the ESPAsyncWebServer library, which handles HTTP and WebSocket connections asynchronously — critical for keeping control latency below ~30ms.

Battery monitoring accounts for the ~2V forward voltage drop across the L298N's internal diodes, giving an accurate reading of actual cell voltage rather than the naive ADC reading. Voltage is smoothed with a rolling average over 8 samples.
